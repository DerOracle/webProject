I experimented a bit with SQLite as shown in the picture above. I had done the very basic in SQL and made a table called "user".
Using the SQLite syntax is pretty simple, first you have to set up sqlite3 dependencies through "npm install sqlite3" and set up a
database which i called "database.db". As for the sqlite3 syntax, it startet with declaring to run a db function then to .run "sql syntax".

So basically what is happening here is that i am calling sqlite3 dependencies through "db", then specifying that i will run a command with ".run".
Then i complete the line with the SQLite syntax ("CREATE TABLE user (values...)")

After the table has been created it's time to add data to it. By declareing a variable (sql in this case) and give it properties of SQL INSERT syntax,
Then at the VALUES write all values as ? within the parantheses. I did this because adding the values as we run the db.run function makes the code
much more clearer and gives us more versatility as to what kind of information we can add, as i will add information into the table as arrays.

Here I'm using an API called Axios which is an easy way to get randomized data which you can adjust to your liking. In this example i am
getting information like first_name, password, uid and id. After I have declared all of the values i wish to add, I then can go back to the db.run (sql, "data")
function and add the variable names in an array as shown in the picture above. Added a err function on the end to check if the data was inserted.

Something to note here is that it's always good to close of the db after you have updated with information,
and add an error check at the end in case of anything wrong in the code.

As for displaying the data that has been added to the database, we need to run a query with the SELECT (dataType, ColumnName, etc) From (table).
Same as before we need to make a variable (sql in this case(comented out the last one so they dont overwrite and cause trouble)) and have it be
SELECT * (* = everything) FROM users (table created previously) then call db.all and sql. Here we also have a function that calls for each row
and console logs it. We also have the err method in case of anything wrong with the code.
